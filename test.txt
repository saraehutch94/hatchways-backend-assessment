Step 3: Testing API routes

Here is where I conducted my tests for my Node.js API.
I deployed my API to Heroku for testing and for you to view
and make requests from as well. The two main routes for this
API are here:

Ping route:
https://hatchway-backend-assignment.herokuapp.com/api/ping

Posts route:
https://hatchway-backend-assignment.herokuapp.com/api/posts

I utilized the REST Client extension on VSCode to
make my requests for this API, as well as Postman. To utilize
the REST Client extension:

1.) Go to the VSCode Extensions tab on the left corner of the 
window and search "REST Client".
2.) Install the extension and create a test.text file
3.) Paste the API routes in which you want to test, and highlight
them
4.) After highlighting, press command + shift + P (Mac),
and search "Rest". You will see "Rest Client: Send Request".
5.) This will bring up results on the right side of your VSCode
window, displaying the response.

For server-side caching of this API, I utilized an npm library
for redis called express-redis-cache. Unfortunately, these caching
routes do not work in the Heroku hosted version. I tried to make
